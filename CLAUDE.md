# 设计原则

## 周易三义 + 第一性原理

所有代码设计和重构决策必须遵循：

### 简易（Simplicity）
- 多个函数做同一件事 → 合并为一个
- 多个状态管同一个概念 → 统一为一个对象
- 代码能删就删，能合就合
- 每加一个功能不应该需要改 N 个地方

### 变易（Change）
- 系统在状态间流转（空闲/加载/播放/跳转/销毁）
- 用状态驱动 UI，不要手动管理每个状态转换
- 事件监听统一注册、统一清理

### 不易（Constancy）
- 变化背后有不变的原则
- 用户意图不因实现细节而改变
- 先抓住本质，再处理表象

### 第一性原理
- 回到最基本的事实，从那里开始推理
- 问自己：如果从零开始设计，会怎么做？
- 不在现有代码上修修补补，而是从本质出发重构

## 代码规范

- Python 3.13，FastAPI + uvicorn
- 前端：原生 JS（无框架），hls.js 播放
- 默认端口 8000（见 memory/project_port.md）
- ffmpeg VideoToolbox 硬件转码（macOS）
- HLS 分片 6 秒，最少 3 分片才开始服务
